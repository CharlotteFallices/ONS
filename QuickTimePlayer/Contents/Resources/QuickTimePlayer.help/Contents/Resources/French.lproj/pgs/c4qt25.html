Création de séquences optimisées pour la transmission sur le web et l’iPhone

Vous pouvez créer facilement des versions de votre séquence optimisées pour l’iPhone et l’affichage sur le bureau. Ceci vous évite de réaliser plusieurs étapes de création de séquences pour la distribution en ligne et vous êtes sûr que votre public reçoit la meilleure qualité d’affichage possible en fonction des appareils utilisés.

Pour exporter pour la transmission sur le web et l’iPhone :

  1. Sélectionnez Fichier > Exporter pour le web.
  2. Dans la boîte de dialogue qui apparaît, indiquez un nom de fichier et sélectionnez un emplacement pour les séquences exportées et le fichier fragment HTML.
  3. Sélectionnez les versions que vous voulez créer.
    • iPhone : crée une séquence optimisée pour la transmission sur l’iPhone par connexion Wi-Fi.
    • iPhone (téléphone cellulaire) : crée une séquence optimisée pour la transmission sur l’iPhone par connexion cellulaire (EDGE).
    • Bureau : crée une séquence optimisée pour la transmission vers des ordinateurs par connexion à haut débit.
  4. Choisissez une option d’affiche pour la séquence.

    L’affiche est l’image fixe que les spectateurs voient lorsqu’ils chargent la page web avec la séquence de référence encodée. Si vous voulez définir une nouvelle affiche avant l’exportation des versions, cliquez sur Annuler. Allez ensuite jusqu’à l’affiche que vous souhaitez utiliser et choisissez Présentation > Définir comme affiche.

  5. Cliquez sur Exporter pour créer les fichiers nécessaires aux options sélectionnées.

    Le processus d’exportation crée plusieurs fichiers, y compris des fichiers pour chaque version de la séquence que vous sélectionnez, une séquence de référence qui fournira automatiquement la bonne version au spectateur et un fichier HTML contenant un fragment de code utilisable pour encoder la séquence sur une page web. Vous devez copier tous ces fichiers sur votre serveur web, au même emplacement correspondant à chacun et avec les mêmes noms de fichier, de façon à ce que la séquence de référence et le fragment HTML fonctionnent correctement.
