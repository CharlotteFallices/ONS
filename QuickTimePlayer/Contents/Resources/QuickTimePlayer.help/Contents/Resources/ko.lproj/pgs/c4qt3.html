비디오 보내기 설정 사용자화하기

보내기를 위한 비디오 설정을 사용자화하려면,

  1. 파일 > 보내기를 선택하십시오.
  2. 보내기 팝업 메뉴에서 파일 포맷을 선택하십시오.
  3. 옵션을 클릭하십시오.
  4. 동영상 설정 대화상자의 비디오 영역에서 설정을 클릭하고, 설정을 선택하십시오.

    압축 종류:  비디오를 압축하는 데 사용하려는 비디오 압축기(코덱)를 선택하십시오. 최저의 데이터 속도(또는 최소 파일)에서 최고의 품질을 위해, H.264를 권장합니다.

    데이터 속도(비트율):  일반적으로, 데이터 속도가 높을수록 품질은 향상되지만, 파일의 크기가 커집니다. 대부분의 경우, 동영상을 나타내는 방식에 따라 데이터 비율을 설정할 수 있습니다 . 예를 들어, 인터넷 전화 연결로 스트리밍하려면 데이터 속도를 초당 약 45Kbits/s로 제한하여 네트워크 트래픽에 대한 여유를 두어야 합니다. 파일이 재생을 위해 다운로드되는 경우, 데이터 속도가 높아질 수 있습니다.(재생이 시작되기 전에 56K 모뎀 사용자는 더 오래 대기해야 합니다.) 동영상의 데이터 비율은 프레임 속도과 같은 사용자가 설정한 다른 압축 옵션에도 영향을 받습니다. 압축기에서 자동으로 선택하여 적절한 데이터 속도가 선택되도록 하려면,

    다음에 최적화:  "다음에 최적화" 팝업 메뉴에서 원하는 전달 방법을 선택하십시오. 이 설정은 코덱이 선택한 데이터 속도의 위로 또는 아래로 얼마나 다양한 데이터 속도가 있는지 구별합니다. 이 옵션은 H.264와 같이 제한을 승인하는 압축기에서만 사용할 수 있습니다.

    키 프레임 옵션:   다수의 압축기는 "프레임 차이"을 사용하여 움직이는 이미지를 압축합니다. 프레임 차이는 어떤 정보를 시작 프레임("키 프레임"으로 불리는)에서 다음 프레임으로 변경할 것인지 정하는 과정입니다. 키 프레임은 이미지의 모든 정보를 포함합니다. 다음 프레임은 변경한 정보만을 포함합니다.

    사용하는 압축기에 따라, 얼마나 자주 키 프레임을 생성할 지를 결정할 수 있습니다. 키 프레임이 충분하지 않다면, 대부분의 프레임이 다른 프레임에서 생성되므로 동영상의 품질이 낮아질 것입니다. 하지만, 키 프레임이 많으면 높은 데이터 비율을 가진 많은 동영상이 될 수 있습니다. 일부 압축기에서 추가된 프레임은 변경한 이미지가 많다면 키 프레임을 자동으로 삽입합니다. 일반적인 사용을 위해 가장 좋은 규칙은 5초 마다 1개의 키 프레임을 가지는 것입니다.(초당 프레임이 5배로 증가됩니다.) RTSP 스트리밍에 대한 파일을 생성하는데 (공용 인터넷과 같은) 전달 네트워크의 신뢰성에 대해 걱정이 되는 경우, 1 또는 2초 마다 1개의 키 프레임으로 키 프레임 빈도수를 늘릴 수도 있습니다. 압축기가 키 프레임 간격을 선택하도록 하려면, 자동을 선택하십시오.

  5. 프레임 속도:  프레임 속도은 매초마다 나타나는 개별 이미지 숫자입니다. 표준 비디오(NTSC)는 29.97fps(초당 프레임)이며, 표준 필름은 24fps입니다. 유렵 표준(PAL)은 25fps입니다. 때때로 QuickTime 동영상은 대역폭과 CPU 필요량을 줄이기 위해 낮은 프레임 속도로 생성됩니다.

    높은 프레임 속도의 동영상은 더 나은 움직임을 보여주지만, 파일의 크기가 커집니다. 동영상의 현재 프레임 속도보다 낮은 프레임 속도을 선택했을 경우, 프레임이 제거될 것입니다. 동영상의 현재 프레임 속도보다 높은 수가 선택되는 경우, 기존의 프레임이 복제될 것입니다.(품질은 향상되지 않고 파일 크기만 커지기 때문에 권장하지 않습니다.) 프레임 속도을 선택할 때, 1/2, 1/3, 1/4및 1/5와 같은 현재 프레임 속도의 간편 분할을 사용하십시오. 예를 들어, 현재 프레임 속도이 30(29.97)이라면, 15나 10을 사용하십시오.

  6. 기타 옵션:  일부 압축기 또는 코덱은 코덱에 특정 옵션을 제공합니다. 압축기(코덱)를 선택한 다음, 모든 특별한 옵션이 나타납니다.
